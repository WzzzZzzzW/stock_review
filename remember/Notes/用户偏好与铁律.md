# 用户偏好与铁律（从一个月对话里反复出现的硬约束）

> 违反这些会被立刻指出。新功能默认遵守。

## 钱 / 模型

- **极度在意成本，能免费就免费、最便宜优先**。原话：「我不要付费的东西，能免费最好」「最便宜的模型就可以」。
- 因此选型：视觉用 GLM-4v-flash（免费），chat 用 deepseek-v4-flash（最便宜）。
- 排斥"用大模型硬跑"花钱的方案，**有数据接口就优先用接口**（如涨停板复盘最终选"方案b 数据接口"而非大模型生成/爬公众号）。

## 数据

- **必须最新、实时**。这是高频痛点，反复强调：「我要的是最新的，实时的数据，剩下的你去解决」「你的数据不是最新的，好好检查数据接口」。
- **每日数据必须留存、不能丢失**（涨停板复盘/市场复盘做成**日历**，按天可回看）。原话：「每天的数据都要留存，不能丢失，我觉得这个数据后面会有用」。
- 数据要持久化——曾因「退出终端后所有数据没了」明确要求落库。

## 交互 / 性能

- **打开即见，不要每次重新加载/生成**：页面要缓存，点开直接看；筛选结果要在筛出时就缓存好。
- **多功能并发后台跑，切换界面不打断生成**（→ 模块级后台任务 store / 全局后台任务中心）。
- **自动补跑**：即使当天没开软件，打开后要自动补齐缺的内容（涨停板/市场复盘/脑库导入等所有板块）。
- 误操作要能**撤回**（持仓删除等）。

## 业务细节

- **持仓数量以 100 股（1手）为单位**，没有 1股2股。
- **搜索栏统一支持三种**：股票代码 / 股票名称 / 名称首字母缩写（如 北方华创 = bfhc）。
- **截图识别走微信手机截图**：常一次发多张，要求**多张一起拖入/上传批量解析**，不是一张张。现价要取**实时**，不用截图里的价。
- A股配色：**红涨绿跌**。
- **建议风格必须“激进且理性”**：所有复盘、持仓、自选、行业和明日动作，都要给唯一裁决和明确动作（进攻 / 保留但不追 / 降级 / 剔除 / 收紧仓位等），不能把正反理由并排丢给用户自己选，不能用模糊中性话术当结论。
- **涨跌幅是结果，不是裁决**：任何模块都不得只凭当日涨跌幅判断强弱或生成买卖动作。个股至少综合趋势、量价、动能、日内承接、相对强弱和风险收益；有数据时继续加入板块共振、资金、事件、基本面和估值。板块至少综合涨幅、上涨广度、资金净流和龙头辨识。缺维度时降低覆盖率与置信度，不能拿单一数字假装完成分析。

## 沟通

- **用中文思考和回复**。曾明确不满：「你为啥最近都用英文思考了」。
- 喜欢先探讨方案再动手（「先不动手，我理想的是……我们先来探讨」），重大改造前对齐思路。
- 经常发截图指 bug；常见历史 bug：复盘页**黑屏/卡死**、请求失败、markdown 没渲染、持仓识别不准。
