# 回测架构

## Summary

`backend/services/backtest_service.py`：按 `hold_days` 个交易日再平衡一次，在候选池上回放规则的数值/形态条件，
按 `sort_field` 取 top_k 等权买入。

## 流程

1. **候选池** `_candidate_pool(rule)`：题材规则→题材成分股；数值/形态规则→沪深300。
2. **取历史** `fetch_history_batch(syms, days_back, progress_cb)`（候选）+ `fetch_index_history(bench)`（基准）。
3. **每期选股** `_pick_at_date`：只用 `_HIST_SUPPORTED` 内的条件过滤
   = `{change_pct, turnover, volume, amount, amplitude, price, volume_ratio}`；
   **PE/PB/总市值在回测里跳过**（历史字段没有，会列入 skipped 提示）。
4. `_do_backtest` 组装 rebalances / nav / stats（total_return_pct、bench_return_pct、win_rate、sharpe、max_drawdown 等）。

## API（api/screen_rule.py，prefix `/api/screen-rule`）

- `POST /rules/{rid}/backtest/start?window_days=&hold_days=&top_k=&benchmark=` → 后台线程
- `GET  /rules/{rid}/backtest/status` → 轮询，`state: running|done|error`，`result.stats` 等

## 已修 bug（2026-06-17）

候选股历史走东财被代理屏蔽→**全空帧→0选股→0再平衡→策略线一条平的0**（基准走 sina 正常，所以只有基准有线）。
修法：`fetch_history_batch` 末尾收集东财取空的 symbol，调 `_baostock_history_batch(...)` 用 baostock 兜底补齐再 merge。
验证：测试规则跑出 24 次再平衡、121 净值点、沪深300全量取到、avg_picks=10。

## Gotcha

空帧 → 0 picks → 平线，是回测「全0表现」的通用征兆，先查取数是不是空的。
