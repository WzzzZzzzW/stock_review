# 数据源与代理坑

## Summary

用户机器/环境里有公司代理（baidu-int.com:8891），**屏蔽东财(eastmoney)和百度**。
这是一堆「数据接口异常/回测全0」类 bug 的根因。任何取数功能都要考虑兜底源。

## 各数据源可达性

| 源 | 状态 | 经由 |
|---|---|---|
| **sina（新浪）** | ✅ 可用 | akshare `stock_zh_index_daily` / `stock_zh_a_daily`；watchlist `_fetch_sina_hq` |
| **eastmoney（东财）** | ❌ 被代理屏蔽 | akshare `stock_zh_a_hist`（历史日K主源） |
| **baidu（百度）** | ❌ 被代理屏蔽 | — |
| **baostock** | ✅ 用户机器可用 | 走自有协议；复盘/持仓/今日都依赖它 |

> ⚠️ 在 Claude 沙箱里 baostock 时好时坏（偶发「登录失败/网络接收错误」），但**用户真机上正常**。
> 所以涉及 baostock 的修复，沙箱跑不通不代表逻辑错，要以用户机器为准。

## 兜底策略（已落地）

- **历史日K**（回测候选股）：东财取空 → baostock 兜底补齐（`stock_data.py:fetch_history_batch`）。
- **实时行情/K线**（AI办公室）：`fetch_quick_batch` 失败/无 today → sina 兜底（`office_tools.py:_quick_row`，标 `_fallback`）。
- **基准指数**：`fetch_index_history` 走 sina，不受影响。

## Details

写新取数逻辑时默认：**别只信东财**。主源东财取空就回落 sina 或 baostock；拿到多少用多少，部分失败不阻塞其他股。
