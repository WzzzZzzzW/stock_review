# 卖出建议体系（持仓逐只卖点诊断）

## Summary

「会买是徒弟，会卖是师傅」。持仓页对每只票给实时卖点诊断。
用户明确要求：**结论由当前走势结构+量能驱动，不是由浮动盈亏驱动**——不要机械收益率档位。

## 后端 services/sell_guidance_service.py

`diagnose(pos, tech, exit_rules)` 喂三样给 AI：
1. 持仓面（成本/现价/盈亏%/持有天数/止损目标）
2. 技术面（MA乖离、RSI、MACD、布林、量比、连涨跌、信号标签）
3. 脑库 exit 类规则

SYSTEM 提示词原则（**改时别破坏**）：
- 判断顺序：用户卖出规则 > 走势结构 > 量能/量价 > 指标(挑1-2个) > 盈亏(只定调止盈/止损)。
- **禁止**「赚X%减仓/亏X%清仓」机械档位。
- `summary` 必须**先讲走势/量能**，不准用「盈利X%/亏损X%」开头。
- 技术面数据缺失时：给「持有」+ 低紧迫度 + 「手动看盘确认」，**不要**仅凭盈亏下清仓/减仓。
- 输出严格 JSON：decision(清仓/减仓/持有/加仓)、urgency、summary、reduce_pct、sell_price、stop_price、reasons、matched_rules、advice。
- 解析失败重试一次，仍失败则**抛错**（绝不静默兜底成「持有」，否则误读成确定性持有很危险）。

## 前端 PortfolioPage.tsx

- 用户选了**盘中自动刷新**：`AUTO_REDIAGNOSE_MS = 5min`、`STALE_GUIDANCE_MS = 6min`。
- `useEffect` 依赖 `[market.open, heldKey]`：开盘且有持仓时，stale 就重诊，并每 5min 轮询重诊（processing 中跳过）。
- 副标题：「以当前走势+量能为主…盘中每5分钟自动重诊」。

## 安全红线

任何卖出/操作建议只是**研判**，绝不替用户下单。买卖让用户自己在券商端做。
