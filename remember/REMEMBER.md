# REMEMBER.md — 股票分析项目记忆库

本记忆库**只服务「股票分析 app」这一个项目**，跟着项目目录走。
核心用途：① 一个对话框上下文快压缩时，换新对话框能直接读这里无缝接上；
② 换 agent 时，另一个 agent 打开本项目也能照着这里操作。

> 任何 agent / 对话框接手本项目，**先读 `Projects/股票分析.md`**（项目全貌 + 当前状态），
> 再按需读 `Notes/`（离散技术事实）。不确定的「代码行为/file:line」一律以当前代码为准，先核对再断言。

---

## Capture Rules

- 只记**对操作本项目有用**的事实：架构、运行方式、端口/路径、数据源与坑、约定、已修过的 bug、进行中的工作。
- 与股票项目无关的内容（如百度合同审查规则）**不进本库**——那是另一条业务线。
- 每次会话结束或上下文将压缩前，把「当前进度 / 下一步」更新到 `Projects/股票分析.md` 的 **## Status** 与 **## Log**。
- 一次性、临时的调试细节不留；留下来的必须是「下个 agent 接手会用到」的。

## Processing

- 新事实优先并入已有 Note，不要散落成大量碎片文件。
- 记录踩过的坑时，连同**根因 + 解决办法**一起写（不只写现象）。
- 涉及命令/路径/端口，写**可直接复制执行**的形式。

## Custom Types

- `runbook` — 可照着跑的操作步骤（启动、重启、构建、验证）。
- `gotcha` — 环境/数据源的坑（如代理屏蔽、baostock 单 socket）。

## Connections

- `Projects/股票分析.md` 是总入口，用 wikilink 串联各 `Notes/`。

## Language

- 简体中文为主；代码/命令/字段名保留原文。

## Templates

- 沿用插件默认 `Templates/`。

## Notes

- A股颜色约定：**红=涨、绿=跌**（与美股相反），任何前端改动都遵守。

## 自动化机制（已配钩子，见 .claude/settings.json）

- **SessionStart**：① 插件自动加载 `Persona.md`；② `scripts/surface_status.sh` 把项目当前进度+阅读顺序推进上下文 → 新对话开场即知干到哪。
- **被动捕捉（passive nudge）**：对话中我会主动建议「要不要记下这条」，用户确认才存。
- **PostToolUse(Edit/Write)**：`scripts/log_change.sh` 自动把每次文件改动追加到 `Journal/当天.md`（操作流水账，不含思路）。
- **收尾蒸馏（半自动）**：用户说「更新记忆」或一段工作结束时，AI 要把**决策/思路/进度/踩坑**蒸馏进 `Projects/股票分析.md`（Status+Log）和当天 `Journal/`。流水账记"动了什么"，蒸馏记"为什么、结论是什么"。
- 深挖整段会话用 `/remember:process`。
